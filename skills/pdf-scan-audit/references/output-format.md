# 결과 보고 표준 포맷

검사 결과는 항상 마크다운 표로 정리한다. 다음 섹션을 순서대로.

## 1. 검사 대상

| 항목 | 값 |
|---|---|
| 파일 | `<basename>` (`<size>`) |
| 책 | `<title>` (`<publisher>` · ISBN `<isbn>`) — ISBN과 책명은 본문에서 추출 가능하면 채움 |
| 총 PDF 페이지 | `<n>` |
| 스캐너 / OCR | `<creator>` / `<producer>` (PDF 메타) |
| 스캔 일시 | `<creationDate>` |

## 2. 핵심 결론

한 문장으로 요약. 예:
- "전반적으로 양호. 누락·순서 오류·회전·잘림 없음."
- "회전 누락 N건, 진짜 페이지 누락 M건 발견. 재스캔 또는 후처리 필요."

PDF↔책 페이지 오프셋이 일관되면 그 사실을 명시 ("PDF=책+28 공식이 전 구간 일관 → 누락 없음").

## 3. 일반 메타데이터 검사

| 검사 항목 | 결과 | 판정 |
|---|---|---|
| 페이지 회전 메타 | `<n>페이지 모두 0°` 또는 `회전된 N페이지: ...` | OK / NG |
| 페이지 방향 | `portrait/landscape` 분포 | OK / NG |
| 빈 페이지 | `<n>건` | OK / NG |
| 텍스트 OCR 누락 | `<n>건` | OK / NG |
| 이미지 없는 페이지 | `<n>건` | OK / NG (스캔본은 NG) |
| 스캔 해상도 (DPI) | `<value>` | OK / NG |
| 이미지↔페이지 잘림 | `<n>건` | OK / NG |
| 페이지 크기 분포 | `<min>~<max>` (`<dominant>`) | OK / △ / NG |

## 4. 책 구조 매핑 (PDF ↔ 책 페이지) — 책류만

| PDF 범위 | 내용 | 책 페이지 |
|---|---|---|
| 1 | 앞표지 | — |
| 2~N | 머리말·목차·표기 설명 | 로마자/없음 |
| N+1~M | 본문 + 찾아보기 | 1~K |
| M+1~끝 | 뒷날개·뒷표지 | — |

## 5. 의심 페이지 시각 재검증

### 회전 의심
| PDF 페이지 | 실제 내용 | 판정 |
|---|---|---|
| ... | ... | OK / 회전 필요 |

### 페이지번호 갭
| PDF 페이지 | OCR 추출 | 실제 헤더 | 판정 |
|---|---|---|---|
| ... | ... | ... | OK / 누락 |

## 6. 종합 진단

| 검사 항목 | 결과 |
|---|---|
| 페이지 누락 | 없음 / N건 (PDF p.X, Y) |
| 페이지 순서 오류 | 없음 / 있음 |
| 회전된 페이지 | 없음 / 있음 |
| 잘린 페이지 / 본문 손실 | 없음 / 있음 |
| 빈 페이지 / 스캔 실패 | 없음 / N건 |
| 스캔 해상도 | `<value>` |
| OCR 정확도 | `<recovered>/<total>` |

## 7. 권장 사항 (선택)

- 즉시 사용 가능 / 재스캔 필요 / 부분 후처리 필요 중 하나
- 결함이 발견되었을 때는 어떤 액션이 필요한지 추상적으로만 제시한다. 예: "p.54, p.98 — 180° 회전 검토", "p.300 — 빈 페이지 삭제 검토". 구체적인 도구·명령은 사용자가 자신이 가진 도구로 결정한다.
